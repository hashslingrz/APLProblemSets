Got a minute? — Try APL!

APL is an array-oriented programming language that will change the way you think about problems and data. With a powerful, concise syntax, it lets you develop shorter programs that enable you to think more about the problem you're trying to solve than how to express it to a computer. You can install a free copy, or try it now by entering an expression (use the language bar above to type the special APL symbols), or clicking one of these expressions, followed by Enter, to see it in action:

What can APL do for you?

Are you a Problem Solver (a domain or subject matter expert with problems to solve) or a Programmer (someone who translates those solutions into a computer-executable format)? Problem Solvers benefit from APL's ability to concisely express advanced concepts without getting bogged down with a lot of computerese syntax. Programmers benefit from APL's productivity and brevity. Shorter programs means quicker development time and less code to maintain and debug.

In either case, concepts that take several lines of code in other languages can often be reduced to a few characters in APL. You'll quickly find that APL's symbols are organized in a logical, mnemonic manner that makes it easy to express advanced concepts clearly and concisely, and makes it easy to learn too. Move to the Learn tab and get started!

Additional features

Utilities

APL has a lot of built-in utilities. They have names beginning with the quad character; ⎕ as in ⎕ts for the current time stamp. Please refer to the categorized listing.

System commands

Additionally, there are so-called system commands which are not code, but perform various management actions. They begin with the closing parenthesis character; ) as in )fns to list your functions.

Library

Finally, we've added the entire dfns library which you can call using ⎕SE.name as in ⎕SE.cal 0 .

About TryAPL

TryAPL is a thin client-side interface for Try It Online, which runs Dyalog APL in a container under Security-Enhanced Linux, one of the Dyalog APL platforms, which also include other Linux distributions (even Raspberry Pi), Microsoft Windows, and macOS.

The Learn tab's lessons are Jupyter Notebooks documents and can be used locally. We welcome submissions of notebooks for inclusion in TryAPL, as well as reports of anomalies, suggestions, criticisms, and comments to tryapl@dyalog.com.

Limitations

The interpreter is a full Dyalog for Linux running in script mode without a session, so one cannot, for example, open an editor window or prompt for input using ⎕ or ⍞. Input is limited to a single line at a time. Each input must run within 1 minute and 128 MB. However, only about 20 MB can be carried over from one execution to the next. The state will be rolled back if a limit is exceeded.

No information is kept server-side or in cookies, so nothing persists between instances.
